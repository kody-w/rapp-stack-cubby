# RAPP glossary

| Term | Narrow local meaning |
|---|---|
| **Actual agent** | Independently loadable BasicAgent Python file with a native manifest. |
| **Artifact chain** | Exact source → cubby → rapplication → cubby egg → installed twin sequence. |
| **Brainstem** | Registry, context, provider tool loop, storage, and loopback HTTP runtime. |
| **Capability wire** | The sole local `POST /chat` request/response contract. |
| **Controller** | Only streamable top-level agent; verifies, hatches, supervises, and aggregates children. |
| **Cubby** | Public deterministic build/storage boundary, not private runtime state. |
| **Direct evidence** | Locally preserved repository-at-head audit fact with locator and classification. |
| **Egg** | This profile's bounded offline rapplication container; other egg families are incompatible. |
| **Estate** | Mapped operator discovery index; not proof of a running twin. |
| **Grail pointer** | Behavioral pointer that supplies no source rights or required working context. |
| **Installed twin** | Private isolated source/workspace/state/process created from verified artifacts. |
| **Leviathan (fleet)** | Mapped coordinator for many enrolled brainstem bodies. |
| **Leviathan (wrapped organism)** | Separate mapped package for one multicellular being. |
| **Metropolis** | Mapped aggregate of public estate discovery metadata. |
| **Neighborhood** | Mapped admitted exchange of signed/optionally sealed Commons events. |
| **Rapplication** | Soul, actual agents, and public application manifests assembled for execution. |
| **RAPPID** | Selected canonical application identity; distinct from transport key and display name. |
| **Skill** | Host guidance/integration material; not an actual RAPP agent. |
| **Twin-chat** | Implemented local signed, replay-safe request/response transport through `/chat`. |

## Status distinctions

| Class | State |
|---|---|
| **Profile requirement** | Use these meanings in new code and documentation. |
| **Implemented now** | Terms describing runtime, agents, controller, and local chat. |
| **Mapped/reference only** | Neighborhood, estate, metropolis, both Leviathan senses, and cloud body. |
| **Unsafe/deprecated** | Inferring compatibility from a shared noun or extension. |
| **Future owner** | Feature owners remain defined in `GAP_REGISTER.md`; vocabulary has no separate owner. |
