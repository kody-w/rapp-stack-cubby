# Gap register

Signed twin-chat, stable public product identity, pinned release trust,
immutable source/dependency staging, deterministic exact artifact builders,
strict descriptor-bound archive verification, SPDX, isolated hatch, installed
integrity, and uninstall are implemented and removed from future ownership.
Controller path containment, stale phase recovery, failure-atomic purge/key
rotation, process-orphan reconciliation, exact source promotion,
product/private-instance separation, verified install adoption, explicit model
propagation, signed-only child launch, and the deterministic controller-only
`/chat` route/CLI are also implemented locally and removed from future
ownership.
The explicit network-free attestation provider, signed installed-byte
SelfTest, fresh doctor/bootstrap, one-command development demo, command
manifest/tutorial harness, and local rollback are implemented and removed
from future ownership. They do not satisfy live/public gates.
The explicit private Copilot token-file contract, self-contained GitHub device
login/refresh, exact-model preflight, controller fixed-argv propagation, and a
content-free `gpt-5.4` completion/tool-loop proof are implemented on this host
and removed from future implementation ownership. That local proof does not
attest a future exact public product or owner messaging route.
The exact-inventory Pages site, six release-aware APIs, deep-route-safe 404,
trusted released-state verifier path, immutable official-action workflows,
protected unchanged-candidate workflow, tag-race checks, public postflight
script, and fail-closed publication scanner are implemented locally and
removed from future implementation ownership. Executing its two signed public
gates remains external.
Remaining release gaps are:

| Gap | Owner | Completion evidence |
|---|---|---|
| `G-RELEASE-COMMIT` final clean exact revision and public attestation | `release-attestation` | Phase B protected-main commit supplied to signed sidecar, immutable tag/prerelease, public GitHub attestation |
| `G-PUBLIC-BYTES` published Store/egg/assets equality | `release-attestation` | downloaded bytes equal local SHA256SUMS |
| `G-PUBLICATION-SCAN-GATE` signed candidate and final scans | release operator / next Pages wiring task | candidate zero-finding signed source/history/Pages/artifact receipt uploaded with the prerelease, then a second signed receipt over public redownload and complete Actions log exports |
| `G-IMESSAGE-LIVE` exact-public-product provider repetition and private enrollment | release operator | Phase C exact public twin repeats the implemented provider gate, then proves FDA/Automation, owner self-chat, first route, restart/sleep-wake |
| `G-PUBLISH` exact-commit release/postflight | `release-attestation` | final SHA, host attestation, downloaded-byte equality |

Global/iMessage authentication, packaging signature/archive verification, deterministic route proof, event replay,
payload/account binding, echo/outbox recovery, terminal supervision, redacted
status, and uninstall safety are implemented locally. Live enrollment,
public artifact attestation, and release/Pages execution remain
with their existing owners; none is claimed by local tests.

`publication-scan` now inventories tracked/untracked/ignored candidates,
traverses canonical branch/tag/configured-remote refs and their reachable Git
blobs, commit identities/messages, and tags with fixed plumbing, recursively
scans supported nested archives, verifies and scans exact Pages and release
assets, and accepts explicit downloaded Actions log ZIPs. It emits only
redacted deterministic receipts and can use the pinned release key. It does
not fetch private runtime roots, public assets, deployed Pages, or Actions
logs; the release/Pages wiring task must supply those explicit public inputs.

Local signed `WORKTREE` bytes and offline end-to-end attestation are proven but
are not release lock values. The current content-free host provider proof is
also not a release lock value. Phase A may retain only the four explicit lock
IDs, including `publication-scan`; Phase B requires the signed candidate scan
asset before creating/re-downloading one unchanged prerelease; Phase C
completes private live proof; Phase D requires the second signed scan before
deploying/promoting that same commit with no rebuild.
`STACK_LOCK.json` stays build-blocked for final release/public SHA values,
supported-host/end-to-end attestation, publication, live Pages proof, and live
owner proof. Local tests and development artifacts do not resolve those gates.
