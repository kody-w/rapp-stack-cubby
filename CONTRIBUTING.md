# Contributing

This project is under construction but no longer context-incomplete.
Contributions should remain small, reviewable, deterministic, and limited to
newly authored MIT code or material that has completed the per-file
provenance process. Start with `AI_CONTEXT.md`; do not fetch an external
repository to supply working context.

## Development

Use Python `>=3.11,<3.12`; the attested profile is Python 3.11. The scaffold
has no runtime dependencies and its tests use `unittest`.

```sh
PYTHON=python3.11 scripts/check.sh
```

The check must pass without installing dependencies or accessing the network.
Use `PYTHON=python3.11 scripts/context-check.sh` while changing profiles,
schemas, decisions, runbooks, status, ownership, or generated indexes.
Use `PYTHON=python3.11 scripts/pages-build.sh` after changing Pages inputs and
`PYTHON=python3.11 scripts/pages-check.sh` before handing off any public
surface or workflow.

## Context changes

Implementation truth moves first: change code and focused tests, then update
the relevant canonical profile, `docs/canon/IMPLEMENTATION_STATUS.md`,
gaps/roadmap, and schemas. Add an ADR for a durable trust or compatibility
decision. Update the inventory in `src/rapp_stack_cubby/context.py` for a new
load-bearing file and regenerate:

```sh
PYTHONPATH=src python3.11 -m rapp_stack_cubby.context --root . --write
PYTHONPATH=src python3.11 -m rapp_stack_cubby.catalog --root . --write
PYTHON=python3.11 scripts/pages-build.sh
```

Review generated bytes before updating `PROVENANCE.json` and
`STACK_LOCK.json` hashes. Keep IDs stable, paths local, the dependency graph
acyclic, and every selected capability routed to tested implementation or a
named future owner. See `docs/operations/CONTEXT_MAINTENANCE.md`.

## Boundaries

- Do not alter census, contract, architecture, provenance, or lock files as a
  side effect of implementation work.
- Do not copy source from the grail pointer or any reference-only repository.
- Do not copy external documentation prose; normalize facts in original words
  and retain only provenance locators in evidence artifacts.
- Do not add real configuration, private fixtures, runtime state, or secrets.
- Keep generated artifacts out of `dist/` until their exact digests and
  provenance are recorded.
- Keep internal agents as agents; do not flatten them into controller tools.

Tests must be deterministic, synthetic, network-free, and must not read or
mutate the user's home directory.
